# 画面遷移一覧

本システムは、フロントエンド（React）とバックエンド（Ruby）構成を採用しているため、ヘッダーナビゲーションを介した「画面切り替え（ルーティング）」と、画面内での「モーダル起動（状態遷移）」を組み合わせて遷移を制御する。

---

## 👤 一般ユーザー（町内会メンバー）の画面遷移

一般ユーザーは、他メンバーの情報をカード形式で参照すること（SCR02・SCR03）と、自身のマイページ管理（SCR04）のみを行うことができる 。自身のマイページからは「退会ボタン」が完全に非表示となるため、削除に関連する遷移やモーダルは起動しない。

| 遷移元画面                 | トリガー（操作）                         | 遷移先画面 / 起動モーダル          | 移行方式           | 備考                                                                                       |
| :------------------------- | :--------------------------------------- | :--------------------------------- | :----------------- | :----------------------------------------------------------------------------------------- |
| **SCR01** ログイン画面     | ログイン認証成功                         | **SCR02** メンバー一覧画面         | 画面遷移           | ログイン後の初期トップ画面。                                                |     |
| **SCR02** メンバー一覧画面 | 他メンバーのカードをクリック             | **SCR03** ユーザー情報詳細モーダル | **モーダル起動**   | 画面遷移せず、その場で詳細をポップアップ表示（参照専用） 。                  |
| **SCR03** 詳細モーダル     | 「閉じる」ボタン押下、または枠外クリック | **SCR02** メンバー一覧画面         | **モーダル閉じる** | モーダルを閉じ、一覧表示状態に戻る。                                                       |
| **SCR02** メンバー一覧画面 | ヘッダー「マイページ」押下               | **SCR04** マイページ画面           | 画面遷移           | 自身のプロフィール確認・編集画面へ移動。                                |
| **SCR04** マイページ画面   | ヘッダー「メンバー一覧」押下             | **SCR02** メンバー一覧画面         | 画面遷移           | 組織図画面に戻る                                   |
| **SCR04** マイページ画面   | 「更新する」ボタン押下（更新成功）       | **SCR04** マイページ画面           | 同一画面内再描画   | 成功メッセージを表示し、最新の編集内容を画面に反映 
| 共通ナビゲーションヘッダー | 「ログアウト」ボタン押下                 | **SCR01** ログイン画面             | 画面遷移           | セッションを破棄し、ログイン前状態に戻る     |

## 👑 管理者（町内会役員）の画面遷移

管理者は、一般ユーザーと同様のカード形式の参照（SCR02）に加え、実務に特化した「名簿管理（SCR05）」の操作権限を持つ。
また、自身のマイページ（SCR04）にのみ「退会する」ボタンが表示され、自分自身のアカウント削除を実行できる 。

### 1. 画面間のナビゲーション（ルーティング）

管理者は共通ヘッダーに「メンバー一覧（カード形式）」と「名簿管理（表形式）」の2つの入り口が用意され、役割に応じていつでも切り替えることができる 。

| 遷移元画面                 | トリガー（操作）             | 遷移先画面                 | 移行方式 | 備考                                                        |
| :------------------------- | :--------------------------- | :------------------------- | :------- | :---------------------------------------------------------- |
| **SCR01** ログイン画面     | ログイン認証成功             | **SCR02** メンバー一覧画面 | 画面遷移 | 一般ユーザーと共通のトップ画面に着地する。   |
| **SCR02** メンバー一覧画面 | ヘッダー「名簿管理」押下     | **SCR05** 名簿管理画面     | 画面遷移 | 管理者専用の実務用ダッシュボードへ切り替え 。 |
| **SCR02** メンバー一覧画面 | ヘッダー「マイページ」押下   | **SCR04** マイページ画面   | 画面遷移 | 管理者自身のマイページへ移動 。                   |
| **SCR05** 名簿管理画面     | ヘッダー「メンバー一覧」押下 | **SCR02** メンバー一覧画面 | 画面遷移 | 組織図画面へ切り替え 。                       |
| **SCR05** 名簿管理画面     | ヘッダー「マイページ」押下   | **SCR04** マイページ画面   | 画面遷移 | 管理者自身のマイページへ移動 。                   |
| **SCR04** マイページ画面   | ヘッダー「名簿管理」押下     | **SCR05** 名簿管理画面     | 画面遷移 | 管理者専用の実務用ダッシュボードへ切り替え。 |
| **SCR04** マイページ画面   | ヘッダー「メンバー一覧」押下 | **SCR02** メンバー一覧画面 | 画面遷移 | 組織図画面へ切り替え 。                       |
| 各画面共通ヘッダー         | 「ログアウト」ボタン押下     | **SCR01** ログイン画面     | 画面遷移 | セッションを破棄してログイン前状態へ。       |

### 2. 名簿管理・ユーザー操作（CRUD）に伴う遷移

名簿管理画面（SCR05）およびマイページ画面（SCR04）を起点とする、登録・代理編集・インライン処理および削除の遷移フローである 。

| 遷移元画面                 | トリガー（操作）                               | 遷移先画面 / 起動モーダル         | 移行方式     | 備考 |
| :------------------------- | :--------------------------------------------- | :------------------------------- | :----------- | :--- |
| **SCR05** 名簿管理画面     | 「新規登録」ボタン押下                         | **SCR06** ユーザー登録画面       | 画面遷移     | 代理登録フォームへ遷移。権限選択項目は表示しない。 |

| **SCR06** ユーザー登録画面 | 「戻る」ボタン押下                             | **SCR05** 名簿管理画面           | 画面遷移     | 登録を行わず名簿管理画面へ戻る。 |
| **SCR05** 名簿管理画面     | 対象行の「編集」ボタン押下                     | **SCR07** ユーザー編集画面       | 画面遷移     | 対象ユーザーの代理編集を行う。 |
| **SCR07** ユーザー編集画面 | 「更新」ボタン押下                             | **SCR05** 名簿管理画面           | 画面遷移     | 更新完了メッセージ表示後、名簿管理画面へ戻る。 |
| **SCR07** ユーザー編集画面 | 「戻る」ボタン押下                             | **SCR05** 名簿管理画面           | 画面遷移     | 編集内容を保存せず名簿管理画面へ戻る。 |
| **SCR05** 名簿管理画面     | 「権限」列のバッジクリック                     | 権限更新警告モーダル             | モーダル起動 | 一般⇔管理者の権限変更確認を行う。 |

| 削除警告モーダル           | 「削除します」押下                             | **SCR05** 名簿管理画面           | 非同期更新   | 対象ユーザーを削除し、完了メッセージ表示後に一覧を再描画する。 |

## 👤 一般ユーザー（町内会メンバー）の遷移フロー

一般ユーザーは、メンバー一覧の参照（SCR02・SCR03）と、自身のマイページ管理（SCR04）にアクセス可能です 。
**マイページ画面からは「退会する」ボタンが完全に非表示（権限制御）となるため、削除用の遷移は発生しません。**

### 1. ログインからトップ画面まで

- **【SCR01】ログイン画面**
  - ➔ `ログイン認証に成功` ➔ **【SCR02】メンバー一覧画面（カード形式）** へ遷移

- **【SCR06】ユーザー登録画面（自己入会）**
  - ➔ `必要項目を入力して「登録」をクリック` ➔ 登録完了のメッセージを表示し、**【SCR01】ログイン画面** へ戻る （※システム側で自動的に一般ユーザー権限として初期登録されます ）
  - ➔ `「戻る」をクリック` ➔ **【SCR01】ログイン画面** へ戻る 

### 2. メンバー一覧（組織図）での操作

- **【SCR02】メンバー一覧画面（カード形式）**
  - ➔ `他メンバーのカードをクリック` ➔ **【SCR03】ユーザー情報詳細モーダル** がその場で起動（参照専用・画面遷移なし）
  - ➔ `ヘッダーの「マイページ」メニューをクリック` ➔ **【SCR04】マイページ画面** へ遷移 

- **【SCR03】ユーザー情報詳細モーダル**
  - ➔ `「閉じる」ボタン、またはポップアップ外枠をクリック` ➔ モーダルを閉じ、**【SCR02】メンバー一覧画面** に戻る

### 3. マイページの管理

- **【SCR04】マイページ画面**
  - ➔ `プロフィール項目を編集し「更新する」をクリック` ➔ 更新完了メッセージを画面上部に表示し、**【SCR04】マイページ画面** を再描画（最新データを反映） 
  - ➔ `ヘッダーの「メンバー一覧」メニューをクリック` ➔ **【SCR02】メンバー一覧画面** へ遷移 

### 4. ログアウト

- **共通ナビゲーションヘッダー**
  - ➔ `「ログアウト」ボタンをクリック` ➔ セッション情報を破棄し、**【SCR01】ログイン画面** へ遷移

## 👑 管理者（町内会役員）の遷移フロー

管理者は、一般ユーザーと同様のカード一覧（SCR02）に加え、実務用のダッシュボードである「名簿管理（SCR05）」の操作権限を持ちます。
また、**管理者自身のマイページ（SCR04）にのみ「退会する」ボタンが表示され、自身の退会処理を行うことができます 。**

### 1. ログインからトップ画面まで

- **【SCR01】ログイン画面**
  - ➔ `ログイン認証に成功` ➔ **【SCR02】メンバー一覧画面（共通トップ画面）** へ遷移 
### 2. ヘッダーナビゲーションによる直接切り替え

管理者は、ヘッダーに配置された専用メニューリンクから、各画面へいつでも直接アクセス（切り替えルーティング）できます。

- **【SCR02】メンバー一覧（カード形式）** ➔ `ヘッダー「名簿管理」をクリック` ➔ **【SCR05】名簿管理画面** へ遷移 
- **【SCR02】メンバー一覧（カード形式）** ➔ `ヘッダー「マイページ」をクリック` ➔ **【SCR04】マイページ画面** へ遷移
- **【SCR05】名簿管理画面（表形式）** ➔ `ヘッダー「メンバー一覧」をクリック` ➔ **【SCR02】メンバー一覧画面** へ遷移 
- **【SCR05】名簿管理画面（表形式）** ➔ `ヘッダー「マイページ」をクリック` ➔ **【SCR04】マイページ画面** へ遷移 
- **【SCR04】マイページ画面** ➔ `ヘッダー「メンバー一覧」をクリック` ➔ **【SCR02】メンバー一覧画面** へ遷移 
- **【SCR04】マイページ画面** ➔ `ヘッダー「名簿管理」をクリック` ➔ **【SCR05】名簿管理画面** へ遷移 

### 3. 名簿管理画面（SCR05）における実務操作（インライン・代理操作）

管理者がメンバーの編集、新規登録、権限のインライン更新、削除（名簿削除）を行う際のフローです。

- **代理ユーザー登録の遷移**
  - **【SCR05】名簿管理画面** ➔ `「新規登録」ボタンをクリック` ➔ **【SCR06】ユーザー登録画面（代理登録用）** へ遷移 （※登録フォームに権限選択項目はありません）
  - **【SCR06】ユーザー登録画面**
    - ➔ `「登録」をクリックして完了` ➔ 登録成功メッセージを表示して、**【SCR05】名簿管理画面** へ戻る（※システム側で一律「一般」権限として追加されます ）
    - ➔ `「戻る」をクリック` ➔ **【SCR05】名簿管理画面** へ戻る

- **代理ユーザー編集の遷移**
  - **【SCR05】名簿管理画面** ➔ `対象行の「編集」ボタンをクリック` ➔ **【SCR07】ユーザー編集画面** へ遷移 （※編集フォームに権限変更項目はありません。ニックネーム、自己紹介はフル編集可能です
  - **【SCR07】ユーザー編集画面**
    - ➔ `「更新」をクリックして完了` ➔ 更新完了メッセージを表示して、**【SCR05】名簿管理画面** へ戻る 
    - ➔ `「戻る」をクリック` ➔ **【SCR05】名簿管理画面** へ戻る

- **F21 権限インライン変更（モーダル警告）**
  - **【SCR05】名簿管理画面** ➔ `「権限」列のバッジ（一般/管理者）をクリック` ➔ **「権限更新の警告モーダル」** がその場で起動 
  - **「権限更新の警告モーダル」**
    - ➔ `「いいえ」をクリック` ➔ 処理を中止し、モーダルを閉じる（画面状態を維持） 
    - ➔ `「はい」をクリック` ➔ バックエンドで権限設定を即座に更新し、モーダルを閉じ、**【SCR05】名簿管理画面** を再描画（バッジが更新されます） 

- **F15 ユーザー削除 / 名簿削除（モーダル警告）**
  - **【SCR05】名簿管理画面** ➔ `アクション列の「削除」ボタンをクリック` ➔ **「削除の警告モーダル」** がその場で起動 
  - **「削除の警告モーダル」**
    - ➔ `「いいえ」をクリック` ➔ 処理を中止し、モーダルを閉じる（画面状態を維持） 
    - ➔ `「削除します」をクリック` ➔ 対象ユーザーデータを物理/論理削除し、モーダルを閉じ、削除完了メッセージを表示して **【SCR05】名簿管理画面** を再描画

## 🛑 特殊遷移：マイページからの退会処理（自身のアカウント削除）

管理者権限を持つユーザーが、自身のマイページ（SCR04）からコミュニティを退会する場合の安全な遷移動線です。

- **【SCR04】マイページ画面（管理者ログイン時）** ➔ `「退会する」ボタン（赤色）をクリック` ➔ **「退会の警告モーダル」** がその場で起動
- **「退会の警告モーダル」**
  - ➔ `「いいえ」をクリック` ➔ 処理を中止し、モーダルを閉じる（画面状態を維持）
  - ➔ `「削除します」をクリック` ➔ 自身のアカウントを削除後、自動的にセッションを完全に破棄して強制ログアウトし、**【SCR01】ログイン画面** へ遷移（完了メッセージ表示）

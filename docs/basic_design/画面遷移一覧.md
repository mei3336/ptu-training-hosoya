# 画面遷移一覧

本システムは、フロントエンド（React）とバックエンド（Ruby）によるSPA（Single Page Application）構成を採用しています。
ヘッダーメニューを介した「画面切り替え（ルーティング）」と、同一画面内で起動する「モーダルポップアップ（状態遷移）」を組み合わせた設計となっています [cite: 17, 60]。

## 👤 一般ユーザー（町内会メンバー）の遷移フロー

一般ユーザーは、メンバー一覧の参照（SCR02・SCR03）と、自身のマイページ管理（SCR04）にアクセス可能です [cite: 16, 17, 18]。
**マイページ画面からは「退会する」ボタンが完全に非表示（権限制御）となるため、削除用の遷移は発生しません [cite: 18, 84]。**

### 1. ログインからトップ画面まで

- **【SCR01】ログイン画面**
  - ➔ `ログイン認証に成功` ➔ **【SCR02】メンバー一覧画面（カード形式）** へ遷移 [cite: 17, 86]
  - ➔ `「新規登録はこちら」をクリック` ➔ **【SCR06】ユーザー登録画面（自己登録）** へ遷移 [cite: 73]

- **【SCR06】ユーザー登録画面（自己入会）**
  - ➔ `必要項目を入力して「登録」をクリック` ➔ 登録完了のメッセージを表示し、**【SCR01】ログイン画面** へ戻る [cite: 28, 61]（※システム側で自動的に一般ユーザー権限として初期登録されます [cite: 18, 83]）
  - ➔ `「戻る」をクリック` ➔ **【SCR01】ログイン画面** へ戻る [cite: 75]

### 2. メンバー一覧（組織図）での操作

- **【SCR02】メンバー一覧画面（カード形式）**
  - ➔ `他メンバーのカードをクリック` ➔ **【SCR03】ユーザー情報詳細モーダル** がその場で起動（参照専用・画面遷移なし） [cite: 17, 86]
  - ➔ `ヘッダーの「マイページ」メニューをクリック` ➔ **【SCR04】マイページ画面** へ遷移 [cite: 16, 18, 84]

- **【SCR03】ユーザー情報詳細モーダル**
  - ➔ `「閉じる」ボタン、またはポップアップ外枠をクリック` ➔ モーダルを閉じ、**【SCR02】メンバー一覧画面** に戻る

### 3. マイページの管理

- **【SCR04】マイページ画面**
  - ➔ `プロフィール項目を編集し「更新する」をクリック` ➔ 更新完了メッセージを画面上部に表示し、**【SCR04】マイページ画面** を再描画（最新データを反映） [cite: 28]
  - ➔ `ヘッダーの「メンバー一覧」メニューをクリック` ➔ **【SCR02】メンバー一覧画面** へ遷移 [cite: 16]

### 4. ログアウト

- **共通ナビゲーションヘッダー**
  - ➔ `「ログアウト」ボタンをクリック` ➔ セッション情報を破棄し、**【SCR01】ログイン画面** へ遷移 [cite: 24, 29]

## 👑 管理者（町内会役員）の遷移フロー

また、**管理者自身のマイページ（SCR04）にのみ「退会する」ボタンが表示され、自身の退会処理を行うことができます [cite: 84]。**

### 1. ログインからトップ画面まで

- **【SCR01】ログイン画面**
  - ➔ `ログイン認証に成功` ➔ **【SCR02】メンバー一覧画面（共通トップ画面）** へ遷移 [cite: 17, 86]

### 2. ヘッダーナビゲーションによる直接切り替え

管理者は、ヘッダーに配置された専用メニューリンクから、各画面へいつでも直接アクセス（切り替えルーティング）できます [cite: 16, 48]。

- **【SCR02】メンバー一覧（カード形式）** ➔ `ヘッダー「マイページ」をクリック` ➔ **【SCR04】マイページ画面** へ遷移 [cite: 16]
- **【SCR05】名簿管理画面（表形式）** ➔ `ヘッダー「メンバー一覧」をクリック` ➔ **【SCR02】メンバー一覧画面** へ遷移 [cite: 16, 17]
- **【SCR05】名簿管理画面（表形式）** ➔ `ヘッダー「マイページ」をクリック` ➔ **【SCR04】マイページ画面** へ遷移 [cite: 16]
- **【SCR04】マイページ画面** ➔ `ヘッダー「メンバー一覧」をクリック` ➔ **【SCR02】メンバー一覧画面** へ遷移 [cite: 16, 17]
- **【SCR04】マイページ画面** ➔ `ヘッダー「名簿管理」をクリック` ➔ **【SCR05】名簿管理画面** へ遷移 [cite: 16, 17]

### 3. 名簿管理画面（SCR05）における実務操作（インライン・代理操作）

管理者がメンバーの編集、新規登録、権限のインライン更新、削除（名簿除外）を行う際のフローです [cite: 17, 83]。

- **代理ユーザー登録の遷移**
  - **【SCR05】名簿管理画面** ➔ `「新規登録」ボタンをクリック` ➔ **【SCR06】ユーザー登録画面（代理登録用）** へ遷移 [cite: 74]（※登録フォームに権限選択項目はありません [cite: 18, 83]）
  - **【SCR06】ユーザー登録画面**
    - ➔ `「登録」をクリックして完了` ➔ 登録成功メッセージを表示して、**【SCR05】名簿管理画面** へ戻る [cite: 28]（※システム側で一律「一般」権限として追加されます [cite: 18, 83]）
    - ➔ `「戻る」をクリック` ➔ **【SCR05】名簿管理画面** へ戻る [cite: 75]

- **代理ユーザー編集の遷移**
  - **【SCR05】名簿管理画面** ➔ `対象行の「編集」ボタンをクリック` ➔ **【SCR07】ユーザー編集画面** へ遷移 [cite: 18, 83]（※編集フォームに権限変更項目はありません。ニックネーム、自己紹介はフル編集可能です [cite: 18, 83]）
  - **【SCR07】ユーザー編集画面**
    - ➔ `「更新」をクリックして完了` ➔ 更新完了メッセージを表示して、**【SCR05】名簿管理画面** へ戻る [cite: 28]
    - ➔ `「戻る」をクリック` ➔ **【SCR05】名簿管理画面** へ戻る [cite: 75]

- **F21 権限インライン変更（モーダル警告）**
  - **【SCR05】名簿管理画面** ➔ `「権限」列のバッジ（一般/管理者）をクリック` ➔ **「権限更新の警告モーダル」** がその場で起動 [cite: 83]
  - **「権限更新の警告モーダル」**
    - ➔ `「いいえ」をクリック` ➔ 処理を中止し、モーダルを閉じる（画面状態を維持） [cite: 83]
    - ➔ `「はい」をクリック` ➔ バックエンドで権限設定を即座に更新し、モーダルを閉じ、**【SCR05】名簿管理画面** を再描画（バッジが更新されます） [cite: 83]

- **F15 ユーザー削除 / 名簿除外（モーダル警告）**
  - **【SCR05】名簿管理画面** ➔ `アクション列の「削除」ボタンをクリック` ➔ **「削除の警告モーダル」** がその場で起動 [cite: 83]
  - **「削除の警告モーダル」**
    - ➔ `「いいえ」をクリック` ➔ 処理を中止し、モーダルを閉じる（画面状態を維持） [cite: 83]
    - ➔ `「削除します」をクリック` ➔ 対象ユーザーデータを物理/論理削除し、モーダルを閉じ、削除完了メッセージを表示して **【SCR05】名簿管理画面** を再描画 [cite: 28, 83]

## 🛑 特殊遷移：マイページからの退会処理（自身のアカウント削除）

管理者権限を持つユーザーが、自身のマイページ（SCR04）からコミュニティを退会する場合の安全な遷移動線です [cite: 84]。

- **【SCR04】マイページ画面（管理者ログイン時）** ➔ `「退会する」ボタン（赤色）をクリック` ➔ **「退会の警告モーダル」** がその場で起動 [cite: 84]
- **「退会の警告モーダル」**
  - ➔ `「いいえ」をクリック` ➔ 処理を中止し、モーダルを閉じる（画面状態を維持） [cite: 84]
  - ➔ `「削除します」をクリック` ➔ 自身のアカウントを削除後、自動的にセッションを完全に破棄して強制ログアウトし、**【SCR01】ログイン画面** へ遷移（完了メッセージ表示） [cite: 24, 28, 29, 84]

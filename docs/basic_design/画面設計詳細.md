# 画面設計詳細

- イメージに関しては、docs\basic_design\アプリモック.htmlのプレビューをご確認ください。

## 1 共通部品

### 1.1 共通UIパーツ

システム全体で共通利用するUI部品のスタイルおよび挙動を定義し、操作性とデザインの一貫性を確保します 。フロントエンド（React）におけるコンポーネント単位の実装基準となります。

#### 共通UI部品仕様

- **ボタン（Button）** 
  - **青色（Primary / 主要アクション）**: ログイン、登録、更新、警告モーダルの「はい」など、処理を実行・確定する操作に利用します 。
  - **赤色（Danger / 危険アクション）**: 削除、退会、警告モーダルの「削除します」など、データの削除や不可逆な操作に利用します 。
  - **グレー色（Secondary / 補助アクション）**: 戻る、キャンセル、ログアウト、警告モーダルの「いいえ」など、処理の中止や安全な離脱操作に利用します 。
- **入力フォーム（Form Input）** 
  - プレースホルダー（入力例）を明示し、入力エラー（バリデーション違反）時は赤色のアウトラインとエラーメッセージをフォーム下部にリアルタイムに表示します 。
- **アイコン画像（アバター）表示エリア**
  - ユーザーアイコンは一律「丸形（`rounded-full`）」の枠線付きで表示します。
  - プロフィール画像をアップロードしていないユーザーに対しては、システム側で用意した一律の「アバタープレースホルダー画像」を表示します。
- **モーダルポップアップ（Modal Dialog）**
  - 画面全体に半透明の黒い背景（オーバーレイ）を敷き、中央に白いダイアログボックスを浮かび上がらせて表示します。
  - モーダル起動中は、背面にある親画面の要素を操作（クリック等）できないようにキーボードフォーカスおよびクリックイベントを制御します。

### 1.2 共通ヘッダー

ログイン状態および利用者の権限（ロール）に基づき、ヘッダーに表示するナビゲーションメニューを動的に切り替えます 。

#### 画面項目

| 項目名           | 種別   | 説明 / 挙動                                                                                                        |
| :--------------- | :----- | :----------------------------------------------------------------------------------------------------------------- |
| **システムロゴ** | リンク | アプリケーションのロゴマークおよび名称を表示する。クリック時はログイン後の初期画面（SCR02）へ遷移する 。 |
| **メンバー一覧** | リンク | コミュニティ交流用のカード形式画面（SCR02）へ遷移する 。【全ユーザー表示】                               |
| **名簿管理**     | リンク | 管理業務用の表形式画面（SCR05）へ遷移する 。**【管理者ログイン時のみ表示】**                             |
| **マイページ**   | リンク | 自身のプロフィールを管理するマイページ画面（SCR04）へ遷移する 。【全ユーザー表示】                       |
| **ログアウト**   | ボタン | クリック時にセッション破棄APIを実行し、未ログイン状態となってログイン画面（SCR01）へ遷移する 。      |

## 2 ログイン画面（SCR01）

### 画面概要

利用者がメールアドレスおよびパスワードを入力し、システムへログインするための認証画面です。
管理者（町内会役員）がログインを試みた場合は、セキュリティ強化のため、一次認証成功後に画面を遷移させず、その場で「二要素認証コード入力モーダル」を起動して追加の認証を行います。

### 画面項目

| 項目名                         | 種別   | 説明 / バリデーション / 挙動                                                                                                                                                                                                     |
| :----------------------------- | :----- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **メールアドレス**             | 入力   | ログイン用メールアドレスを入力する （必須、メールアドレス形式チェック ）。                                                                                                                                   |
| **パスワード**                 | 入力   | ログイン用パスワードを入力する （必須、非表示マスク入力）。                                                                                                                                                            |
| **ログインボタン**             | ボタン | 認証APIを非同期で呼び出す 。<br>・成功：対象者の権限に関わらず「SCR02 メンバー一覧画面」へ遷移する 。<br>・失敗：エラーメッセージを表示する 。連続5回失敗でアカウントをロックする 。 |
| **エラーメッセージ表示エリア** | 表示   | 認証失敗時、またはアカウントロック時に適切なメッセージを表示する 。                                                                                                                                                    |

### 画面内モーダル：二要素認証コード入力モーダル（管理者専用）

ログインしたアカウントが管理者権限（`role: 1`）だった場合にのみ、一次認証成功時にログイン画面上にオーバーレイ表示されるワンタイムパスワード確認用のダイアログです。

| 項目名                         | 種別   | 説明 / バリデーション / 挙動                                                                                                                                                                                                                         |
| :----------------------------- | :----- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **認証コード入力欄**           | 入力   | スマートフォンの認証アプリ（Google Authenticator等）に表示される現在の6桁の数字を入力するフォーム（必須、半角数字6桁制限、プレースホルダー「000000」）。                                                                                             |
| **認証するボタン**             | ボタン | TOTP検証APIを非同期で呼び出す（主要アクション・青色）。<br>・**検証成功**：正式にセッションを確立し、「SCR02 メンバー一覧画面」へ遷移する 。<br>・**検証失敗**：モーダル内のエラー表示エリアに「認証コードが正しくありません」と表示する。 |
| **キャンセルボタン**           | ボタン | 認証処理を中断してモーダルを閉じ、通常のログイン入力状態に戻る（補助アクション・グレー） 。セッションは確立されない。                                                                                                                       |
| **モーダル内エラー表示エリア** | 表示   | コード未入力時、または検証失敗（コード不正）時に、モーダル内にリアルタイムに適切なメッセージを表示する。                                                                                                                                             |

## 3 メンバー一覧画面（カード形式 / SCR02）

### 画面概要

一般ユーザー・管理者の双方がアクセス可能なログイン後の共通トップ画面です 。町内会メンバーの「お互いの顔や個性を知る」ことを目的とした組織図（ギャラリービュー）のレイアウトを採用します 。

### 画面項目

| 項目名                           | 種別 | 説明 / 挙動                                                                                                                                                                                                           |
| :------------------------------- | :--- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **名前検索（フィルター）入力欄** | 入力 | 「氏名」または「ニックネーム」の一部を入力するテキストエリア 。<br>1文字入力されるごとに、React側の状態（State）を更新し、**カード一覧をリアルタイムで絞り込み（フィルター）表示**する 。 |
| **メンバーカードグリッド**       | 表示 | 3列（またはレスポンシブな複数列）のカードレイアウトで、全登録メンバーを一覧表示する 。                                                                                                                  |
| **（カード内）アイコン画像**     | 表示 | 登録されたプロフィール画像を表示する（未設定時はデフォルトアバター） 。                                                                                                                                     |
| **（カード内）ニックネーム**     | 表示 | ユーザーのニックネームを表示する（未設定の場合は、登録された「氏名」を表示する）。                                                                                                                                    |
| **（カード内）自己紹介（bio）**  | 表示 | 登録された自己紹介文を表示する。スペースに合わせて一定文字数（例: 40文字）を超えた部分は「...」で省略表示する。                                                                                                       |

### 画面挙動

1.  **詳細モーダルの起動**: メンバーカードのいずれかの領域（カード全体）をクリックした際、対象ユーザーの詳細情報をパラメーターとして渡し、画面を遷移させずに「SCR03 ユーザー情報詳細モーダル」をその場でポップアップ起動させます 。

## 4 ユーザー情報詳細モーダル（SCR03）

### 画面概要

メンバー一覧画面（SCR02）からカードをクリックした際にポップアップ表示される、他メンバーのプロフィール情報の参照専用モーダルです 。編集処理や削除処理の導線は一切排除した「シンプルな閲覧専用画面」とします。

### 画面項目

| 項目名              | 種別   | 説明 / 表示内容                                                                                             |
| :------------------ | :----- | :---------------------------------------------------------------------------------------------------------- |
| **アバター画像**    | 表示   | アップロードされたプロフィール画像を表示する（拡大表示） 。                                       |
| **氏名**            | 表示   | 対象ユーザーの「氏名」を大きく表示する。                                                                    |
| **ニックネーム**    | 表示   | 登録がある場合は、氏名の下部に括弧や専用スタイルでニックネームを併記する。                                  |
| **自己紹介（bio）** | 表示   | 登録された自己紹介文（bio）の全文を改行を維持して表示する。                                                 |
| **閉じるボタン**    | ボタン | ダイアログの右上「×」ボタン、または右下の「閉じる」ボタン（グレー色） 。押下時にモーダルを閉じる。 |

---

## 5 マイページ画面（SCR04）

### 画面概要

自身のプロフィール情報（氏名、メール、パスワード、ニックネーム、自己紹介、アイコン画像）を確認・変更するための自分専用の管理画面です 。**一般ユーザーがアクセスした際は退会ボタンを非表示とし、管理者のみに退会ボタンを表示します（完全な権限制御）** 。

### 画面項目

| 項目名                       | 種別   | 説明 / バリデーション / 挙動                                                                                                                                                                                                              |
| :--------------------------- | :----- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **現在のアイコンプレビュー** | 表示   | 現在設定されているプロフィール画像を表示する 。                                                                                                                                                                                 |
| **アイコン画像選択**         | 入力   | 新しい画像ファイルを選択するファイルフォーム（`<input type="file">`）。<br>・制限仕様：**形式（JPEG/PNGのみ）、容量（1ファイル最大2MBまで）**。不適合時はエラーを表示する。                                                               |
| **氏名**                     | 入力   | 氏名の編集用テキスト欄 （必須、最大50文字 ）。                                                                                                                                                                        |
| **メールアドレス**           | 入力   | メールアドレスの編集用テキスト欄 （必須、最大255文字、メール形式、重複チェック ）。                                                                                                                                   |
| **パスワード**               | 入力   | 変更時のみ入力するパスワード用の欄。空欄の場合はパスワードを更新しない仕様とする（任意、8〜32文字、英大文字・小文字・数字混在 ）。                                                                                          |
| **ニックネーム**             | 入力   | ニックネームの編集用テキスト欄 （任意、最大15文字 ）。                                                                                                                                                                |
| **自己紹介（bio）**          | 入力   | 自己紹介（bio）の編集用複数行テキスト欄（任意、最大200文字程度）。                                                                                                                                                                        |
| **更新するボタン**           | ボタン | 編集されたプロフィール情報の更新APIを送信（Multipart Form送信） 。<br>・成功時：サクセスメッセージを画面に表示し、画面データを再読み込みする 。                                                                       |
| **退会するボタン**           | ボタン | **【重要】一般ユーザーログイン時は完全に非表示（DOM上に配置しない）** 。<br>**「管理者権限」のユーザー自身のマイページにのみ赤色ボタンとして表示する。**<br>ボタンクリック時に「退会の警告モーダル」を起動する 。 |

### 画面挙動

1.  **退会の警告モーダル（F11）**: 管理者が「退会する」をクリックした際に起動するダイアログ 。
    - **表示文言**: 「本当に退会しますか？ 退会すると、あなたのアカウント情報は会員名簿から完全に削除され、復元することはできません。」
    - **ボタン配置**:
      - **「いいえ」**（グレー色ボタン）：処理を中止しモーダルを閉じる 。
      - **「削除します」**（赤色ボタン）：アカウント削除APIを実行し、完了後セッションを破棄してログイン画面（SCR01）へ強制遷移する 。

## 6 名簿管理画面（表形式 / SCR05）

### 画面概要

管理者（町内会役員）が名簿のメンテナンスや権限設定などの事務作業を効率的に行うためのダッシュボード画面（テーブルリスト形式）です 。詳細画面への無駄な遷移を廃止し、この画面上のみで主要な管理業務が完結する設計とします 。

### 画面項目

| 項目名                           | 種別   | 説明 / 挙動                                                                                                                                                                      |
| :------------------------------- | :----- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **名前検索（フィルター）入力欄** | 入力   | 氏名・ニックネームを入力して、**テーブル名簿をリアルタイムに絞り込み表示**する検索フォーム 。                                                                      |
| **新規登録ボタン**               | ボタン | ユーザー登録画面（SCR06）への遷移ボタン（青色） 。                                                                                                                  |
| **ユーザー一覧テーブル**         | 表     | 全ユーザーの情報を表示するテーブル。詳細画面への遷移リンクは設けない 。<br>・列構成：`ID`、`氏名`、`メールアドレス`、`ニックネーム`、`権限`、`操作`          |
| **権限（インラインボタン）**     | 列項目 | ユーザーの現在の権限を「管理者（赤バッジ）/ 一般（青バッジ）」のバッジ形式ボタンで表示 。<br>クリック時に「F21 権限更新の警告モーダル」をその場で起動する 。 |
| **操作（編集ボタン）**           | 列項目 | クリック時に、対象ユーザーの代理編集画面（SCR07）へ遷移する 。                                                                                                         |
| **操作（削除ボタン）**           | 列項目 | クリック時に、「F15 削除の警告モーダル」をその場で起動する 。                                                                                                          |

### 画面挙動

1.  **F21 権限更新の警告モーダル**: 権限バッジをクリックした際にその場で起動するダイアログ 。
    - **表示文言**: 「本当に権限を変更しますか？」 
    - **ボタン配置**:
      - **「いいえ」**（グレー色）：処理を中断し、変更せずにモーダルを閉じる 。
      - **「はい」**（青色）：権限（admin ⇔ user）を変更するAPIを実行し、即座にテーブルを再描画する 。
2.  **F15 削除の警告モーダル**: アクション列の「削除」をクリックした際にその場で起動するダイアログ 。
    - **表示文言**: 「本当にこのユーザーを削除（名簿から削除）しますか？」
    - **ボタン配置**:
      - **「いいえ」**（グレー色）：処理を中断し、削除せずにモーダルを閉じる 。
      - **「削除します」**（赤色）：物理削除（または論理削除）APIを実行し、完了メッセージを表示してテーブルを再描画する 。

## 7 ユーザー登録画面（SCR06）

### 画面概要

新規にユーザー（町内会メンバー）を登録するための画面です 。一般ユーザー自身による「自己入会（自己登録）」と、管理者による「メンバーの代理追加（代理登録）」の双方で共通フォームを使用します 。
**入力の簡略化とミス防止のため、「権限（一般/管理者）」を選択するフィールドは画面から完全に廃止します（システム側で一律「一般ユーザー」として登録し、管理者は登録完了後にSCR05から昇格させます）** 。

### 画面項目

| 項目名             | 種別   | 説明 / バリデーション / 挙動                                                                                                                                              |
| :----------------- | :----- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **氏名**           | 入力   | 登録するユーザーの氏名 （必須、最大50文字 ）。                                                                                                        |
| **メールアドレス** | 入力   | ログインに使用するメールアドレス （必須、最大255文字、メール形式、重複チェック ）。                                                                   |
| **パスワード**     | 入力   | ログイン時に使用するパスワード （必須、非表示マスク入力、8〜32文字、英大文字・小文字・数字混在 ）。                                               |
| **登録ボタン**     | ボタン | 登録を実行する（青色） 。成功時は完了メッセージを表示し、遷移元画面（自己登録時はログインSCR01、管理者登録時は名簿管理SCR05）へ遷移する 。 |
| **戻るボタン**     | ボタン | 登録処理を行わずに前画面（自己登録時はログインSCR01、管理者登録時は名簿管理SCR05）へ戻る（グレー色） 。                                                      |

## 8 ユーザー編集画面（SCR07）

### 画面概要

管理者が名簿管理画面（SCR05）から他メンバーのプロフィール情報を代理でメンテナンス・修正するための編集画面です 。
**マイページ編集画面と同様、画面から「権限」の変更項目を削除し、純粋なプロフィール情報（ニックネームや自己紹介：bioを含む全項目）をフル編集・保存することに特化させます** 。

### 画面項目

| 項目名                       | 種別   | 説明 / バリデーション / 挙動                                                                                                                          |
| :--------------------------- | :----- | :---------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ユーザーID**               | 表示   | 対象ユーザーのID（読取専用・変更不可） 。                                                                                                   |
| **現在のアイコンプレビュー** | 表示   | 対象ユーザーの現在のプロフィール画像を表示する 。                                                                                           |
| **アイコン画像選択**         | 入力   | 代理で新しい画像ファイルを選択するファイルフォーム（`<input type="file">`）。<br>・制限仕様：**形式（JPEG/PNGのみ）、容量（1ファイル最大2MBまで）**。 |
| **氏名**                     | 入力   | 氏名の編集用テキスト欄 （必須、最大50文字 ）。                                                                                    |
| **メールアドレス**           | 入力   | メールアドレスの編集用テキスト欄 （必須、最大255文字、メール形式、重複チェック ）。                                               |
| **パスワード**               | 入力   | パスワードの強制上書き変更用の欄（未入力の場合は変更しない仕様）。（任意、8〜32文字、英大文字・小文字・数字混在 ）。                    |
| **ニックネーム**             | 入力   | ニックネームの編集用テキスト欄 （任意、最大15文字 ）。ニックネーム・自己紹介はフルで編集・保存が可能です 。             |
| **自己紹介（bio）**          | 入力   | 自己紹介（bio）の編集用複数行テキスト欄 （任意、最大200文字程度）。                                                                         |
| **更新ボタン**               | ボタン | 代理での更新処理を実行する（青色） 。成功時は完了メッセージを表示し、名簿管理画面（SCR05）へ遷移する 。                    |
| **戻るボタン**               | ボタン | 更新を行わずに、名簿管理画面（SCR05）へ戻る（グレー色） 。                                                                               |
```

# 1. システムの目的

本システムは、PTU研修STEP1においてログイン機能とユーザー管理機能の開発を通じて、要件定義から設計、実装、試験までの一連の開発工程を学ぶための研修用システムである。

本研修の目的は、プログラミング技術そのものを学ぶことではなく、実際のシステム開発の流れを理解し、要件定義から設計、実装、試験、結果報告までを一通り経験することで、SI（システムインテグレーション）の工程規律を身につけることである。

また、AIによる支援を受ける前提の時代であっても、

- 自分自身で仕様を理解する
- 作成した成果物を説明できる
- 問題点を発見し修正できる

状態を目指す。

そのため、本システムは単なるログイン機能の実装を目的とするのではなく、「読む・直す・説明する力」を養い、今後の開発業務において主体的に成果物へ責任を持てるようになるための実践経験を積むことを目的とする。

---

# 2. 業務要件

本システム導入後、利用者は認証を行った上でユーザー情報を管理できる。

想定する業務フローは以下の通りとする。

## 一般ユーザー

1. 利用者がログインする
2. システムが認証を行う
3. ログイン成功後、ユーザー画面へ遷移する
4. ユーザー情報の参照・編集・削除を行う
   - 削除を行った場合は強制ログアウトする
5. 利用終了後にログアウトする

## 管理者

1. 管理者がログインする
2. システムが認証を行う
3. ログイン成功後、ユーザー管理画面へ遷移する
4. ユーザー情報の参照・編集・削除を行う
5. ユーザー権限を編集する
6. 利用終了後にログアウトする

本システムを通じて、ユーザー認証からデータ管理までの基本的な業務プロセスを体験できることを要件とする。

# 3. 機能要件

## 3.1 ログイン機能

### 認証機能

- ユーザーIDとパスワードによるログイン
- ログアウト機能

### パスワード管理

- パスワードのハッシュ化保存

### セッション管理

- ログイン状態の維持
- 未認証ユーザーのアクセス制御

### セキュリティ制御

- ログイン失敗回数の記録
- 一定回数以上失敗した場合の制限

---

## 3.2 ユーザー管理機能

### 一般ユーザー

- 自身のユーザー情報参照
- 自身のユーザー情報編集
- 自身のユーザー情報削除

### 管理者

- ユーザー一覧表示
- ユーザー登録
- ユーザー編集
- ユーザー削除
- 権限設定

### 共通ルール

#### 入力チェック

- 必須項目チェック
- 文字数チェック
- 形式チェック

#### 重複チェック

- 同一ユーザーID登録防止

#### 権限制御

- 一般ユーザーは自身の情報のみ操作可能とする
- 管理者は全ユーザー情報を操作可能とする

---

# 4. 非機能要件

## 性能

- 一般的な操作において画面遷移が大きな待ち時間なく行えること

## セキュリティ

- パスワードはハッシュ化して保存する
- 認証されていない利用者はユーザー管理機能へアクセスできない
- 一般ユーザーは自身のユーザー情報のみ操作可能とする
- 管理者のみ全ユーザー情報の登録・参照・編集・削除を行えるものとする

## 可用性

- ブラウザを通じて利用できること

## 保守性

- ソースコードの可読性を確保する
- 機能追加・修正が行いやすい構造とする

# 3. シーケンス図
## 3.1 ログイン
### 基本ログイン
メンバー　*メンバー権限の場合

```mermaid
sequenceDiagram

    actor メンバー
    participant SCR01 as ログイン画面
    participant Auth as 認証処理
    participant DB as users

    メンバー->>SCR01: メールアドレス・パスワード入力
    SCR01->>Auth: ログイン要求
    Auth->>DB: ユーザー情報取得
    DB-->>Auth: ユーザー情報返却

    Auth->>Auth: パスワード照合

    alt 認証成功
        Auth->>Auth: セッション生成
        Auth-->>SCR01: 認証成功
        SCR01-->>メンバー: SCR02へ遷移
    else 認証失敗
        Auth-->>SCR01: エラー返却
        SCR01-->>メンバー: エラーメッセージ表示
    end
```

### 管理者二要素認証 
管理者（二要素認証）*管理者権限の場合   
```mermaid
sequenceDiagram

    actor 管理者
    participant SCR01 as ログイン画面
    participant Auth as 認証処理
    participant OTP as 二要素認証処理
    participant DB as users

    管理者->>SCR01: メールアドレス・パスワード入力
    SCR01->>Auth: ログイン要求

    Auth->>DB: ユーザー情報取得
    DB-->>Auth: ユーザー情報返却

    Auth->>Auth: パスワード照合

    alt 認証失敗
        Auth-->>SCR01: 認証失敗
        SCR01-->>管理者: エラーメッセージ表示

    else 認証成功

        Auth->>Auth: 仮セッション発行
        Auth-->>SCR01: 一次認証成功

        SCR01-->>管理者: 認証コード入力画面表示

        管理者->>SCR01: 認証コード入力
        SCR01->>OTP: 認証コード検証要求

        OTP->>DB: OTP秘密鍵取得
        DB-->>OTP: OTP秘密鍵返却

        OTP->>OTP: 認証コード照合

        alt OTP認証成功

            OTP->>Auth: 本セッション発行要求
            Auth->>Auth: 本セッション発行

            Auth-->>SCR01: ログイン完了
            SCR01-->>管理者: SCR02へ遷移

        else OTP認証失敗

            OTP-->>SCR01: 認証失敗
            SCR01-->>管理者: エラーメッセージ表示
    　　end
    end
```

## 3.3 ユーザー登録
```mermaid
sequenceDiagram

    actor 管理者
    participant SCR06 as ユーザー登録画面
    participant control as UserController
    participant DB as users

    管理者->>SCR06: 登録情報入力
    SCR06->>control: 登録要求

    control->>control: 入力チェック
    control->>DB: メールアドレス重複確認

    alt 重複なし
        control->>control: パスワードハッシュ化
        control->>DB: usersレコード作成
        control->>DB: role=member設定

        DB-->>control: 登録完了

        control-->>SCR06: 登録完了
        SCR06-->>管理者: SCR05へ遷移

    else 重複あり
        control-->>SCR06: エラー返却
        SCR06-->>管理者: エラーメッセージ表示
    end
```

## 3.8 自己退会
管理者権限
```mermaid
sequenceDiagram

    actor 管理者
    participant SCR04 as マイページ画面
    participant control as UserController
    participant DB as users

    管理者->>SCR04: 退会ボタン押下

    SCR04-->>管理者: 退会確認モーダル表示

    管理者->>SCR04: 退会ボタン選択

    SCR04->>control: 管理者数確認要求

    control->>DB: 管理者数取得
    DB -->> control: 管理者数送信

    alt 管理者が2名以上

        control->>DB: 自身のアカウント削除

        control->>control: セッション破棄

        control-->>SCR04: 退会完了

        SCR04-->>管理者: SCR01へ遷移

    else 管理者が1名のみ
    　　control -->> SCR04:組織退会確認モーダル表示依頼
 
        SCR04-->>管理者: 組織退会確認モーダル表示

        管理者->>SCR04: 組織退会実行

        SCR04->>control: 組織退会要求

        control->>DB: 全ユーザーデータ削除（自身を除く）
        DB -->> control: 全ユーザーデータ削除完了通知

        control->>control: システム初期状態へ復元
        control->>DB: 初期管理者アカウント作成
        DB -->> control: 初期管理者アカウント作成完了通知

        control->>DB: 自身のアカウント削除
        DB -->> control: 自身のアカウントデータ削除完了通知

        control->>control: セッション破棄

        control-->>SCR04: 組織退会完了

        SCR04-->>管理者: SCR01へ遷移

    end
```
